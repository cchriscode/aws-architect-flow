# Go for DevOps

> In the DevOps world, Go is the "native language of cloud native." Docker, Kubernetes, Terraform, Prometheus — these core tools are all built with Go. While [Python automation](./02-python) is a versatile scripting tool, Go specializes in **high-performance CLI tools and Kubernetes ecosystem extension**. It compiles to a single binary, handles concurrency easily with goroutines, and creates stable infrastructure tools with a strong type system.

---

## 🎯 Why Do You Need to Know Go?

### Daily Analogy: Swiss Army Knife vs Power Drill

If [Python](./02-python) is a "Swiss Army knife" like a versatile tool, then Go is a **"power drill."** It's optimized for specific tasks, and once compiled (charged), it works anywhere without power.

- Python scripts require "Is Python installed on this server?" check
- Go tools need **just copy a single binary** (like kubectl, terraform!)
- No dependency worries when deploying to 1000 servers

### Real-world Moments When You Need Go

```

```

All core cloud-native tools are built with Go:

### Go's Strengths

**Single Binary** — Zero dependencies
**goroutines** — Lightweight concurrency
**Static Typing** — Compile-time safety
**Cross-compile** — linux/amd64, arm64, etc.
**Fast Build** — Quick execution

### Python vs Go Comparison (DevOps perspective)

| Comparison | Python | Go |
|-----------|--------|-----|
| **Deployment** | Interpreter + packages required | Single binary |
| **Speed** | Slow (interpreted) | Fast (native compiled) |
| **Concurrency** | asyncio / threading (GIL) | goroutine (true concurrency) |
| **Type System** | Dynamic (runtime errors) | Static (compile-time errors) |
| **K8s Ecosystem** | kubernetes library | client-go (first-class citizen) |
| **Learning Curve** | Low | Medium |
| **Main Use** | Scripting, automation, ML | CLI tools, system programming |
| **Representative Tools** | Ansible, AWS CLI | Docker, K8s, Terraform |

> **Core Point**: Python is optimal for "quickly writing automation scripts," Go for "developing deployable infrastructure tools." Knowing both makes you unstoppable!

## 🧠 Grasping Core Concepts

### 1. Go's Design Philosophy

Go was created at Google in 2009. The core philosophy is **"Simplicity."**

- Only 25 keywords (Java has 50+)
- No inheritance — uses composition only
- No exceptions — returns errors as values
- Generics added only in 1.18 (minimal)

Go's Design Principles:
```
"Less is more"
• Fewer features mean more consistent code
• All Go code looks similar — easy to read
• No convention debates (gofmt enforces it)
```

### 2. Go Core Components at a Glance

```
Go Core Components:

Basic Types:         string, int, bool, float64, byte
Struct:              Data bundling (like Python class)
Interface:           Behavior contract (duck typing)
Error Handling:      error return values (if err != nil)
Goroutine:           go func() — lightweight thread
Channel:             ch <- data (goroutine communication)
```

DevOps Applications:
```
CLI Tools:           cobra framework
HTTP Server/Client:  net/http
K8s Client:          client-go
Operator:            controller-runtime
```

### 3. Go Module System

Like Python's pip/virtualenv, Go has a dependency management system.

```
Go Modules Structure:

my-cli-tool/
├── go.mod              ← Like pip's requirements.txt
├── go.sum              ← Dependency hash (integrity check)
├── main.go             ← Entry point
├── cmd/                ← CLI commands
│   ├── root.go
│   ├── deploy.go
│   └── status.go
├── internal/           ← Internal packages (no external import)
│   ├── k8s/
│   └── aws/
└── pkg/                ← Public packages (external import ok)
    └── config/
```

## 🔍 Understanding Each Topic in Detail

### 1. Go Basics — Variables, Types, Functions

```go
// === Go Basic Syntax ===
package main    // Every Go file belongs to a package

import (
    "fmt"       // Output (like Python's print)
    "strings"   // String processing
    "os"        // OS-related (environment variables, files)
)

func main() {
    // --- Variable declaration ---

    // Method 1: var keyword (explicit type)
    var name string = "devops-engineer"
    var age int = 5        // Years of experience
    var isReady bool = true

    // Method 2: := short declaration (type inference — most used!)
    cluster := "production"
    replicas := 3
    cpuLimit := 0.5    // Inferred as float64

    fmt.Println(name, age, isReady)
    fmt.Printf("Cluster: %s, Replicas: %d, CPU: %.1f\n",
        cluster, replicas, cpuLimit)

    // --- Arrays/Slices (like Python list) ---
    regions := []string{"ap-northeast-2", "us-east-1", "eu-west-1"}
    regions = append(regions, "ap-southeast-1")   // Add
    fmt.Println("Regions:", regions)
    fmt.Println("First:", regions[0])             // Indexing
    fmt.Println("Count:", len(regions))           // Length

    // --- Maps (like Python dict) ---
    labels := map[string]string{
        "app":     "web-server",
        "env":     "production",
        "team":    "platform",
    }
    labels["version"] = "v2.1.0"    // Add

    for key, value := range labels {
        fmt.Printf("  %s: %s\n", key, value)
    }

    // --- Read environment variables ---
    home := os.Getenv("HOME")
    if home == "" {
        home = "/root"    // Default value
    }
    fmt.Println("HOME:", home)

    // --- String processing ---
    image := "nginx:1.25-alpine"
    parts := strings.Split(image, ":")
    fmt.Printf("Image: %s, Tag: %s\n", parts[0], parts[1])
}
```

### 2. Structs and Methods

```go
// === Struct — Go's Core Data Type ===
// Similar to Python's class but no inheritance
// Combines "data + behavior"

package main

import (
    "fmt"
    "time"
)

// --- Struct definition ---
type Pod struct {
    Name      string            `json:"name" yaml:"name"`
    Namespace string            `json:"namespace" yaml:"namespace"`
    Image     string            `json:"image" yaml:"image"`
    Replicas  int               `json:"replicas" yaml:"replicas"`
    Labels    map[string]string `json:"labels" yaml:"labels"`
    CreatedAt time.Time         `json:"created_at" yaml:"createdAt"`
}

// --- Method definition (attach function to struct) ---
// (p Pod) — value receiver (original not changed)
func (p Pod) FullName() string {
    return fmt.Sprintf("%s/%s", p.Namespace, p.Name)
}

// (*p Pod) — pointer receiver (original can be changed)
func (p *Pod) Scale(newReplicas int) {
    fmt.Printf("Scaling %s: %d → %d\n", p.Name, p.Replicas, newReplicas)
    p.Replicas = newReplicas
}

func (p Pod) IsOlderThan(duration time.Duration) bool {
    return time.Since(p.CreatedAt) > duration
}

func main() {
    // Create struct
    pod := Pod{
        Name:      "web-server",
        Namespace: "production",
        Image:     "nginx:1.25",
        Replicas:  3,
        Labels: map[string]string{
            "app": "web",
            "env": "prod",
        },
        CreatedAt: time.Now(),
    }

    fmt.Println("Pod:", pod.FullName())     // production/web-server
    pod.Scale(5)                             // Scaling web-server: 3 → 5
    fmt.Println("New replicas:", pod.Replicas) // 5
}
```

### 3. Error Handling — Go's Signature

```go
package main

import (
    "fmt"
    "os"
)

// Go uses errors as return values, not exceptions
func ReadConfig(path string) (map[string]string, error) {
    // Check if file exists
    if _, err := os.Stat(path); err != nil {
        return nil, fmt.Errorf("config file not found: %w", err)
    }

    // Simulate reading config
    config := map[string]string{
        "env": "production",
    }
    return config, nil
}

func main() {
    config, err := ReadConfig("/etc/myapp/config.yaml")
    if err != nil {
        fmt.Printf("Error reading config: %v\n", err)
        return
    }
    fmt.Println("Config:", config)
}
```

### 4. Interfaces — Duck Typing in Go

```go
package main

import "fmt"

// Define interface (contract)
type Logger interface {
    Info(msg string)
    Error(msg string)
}

// Implement for console
type ConsoleLogger struct{}

func (cl ConsoleLogger) Info(msg string) {
    fmt.Printf("[INFO] %s\n", msg)
}

func (cl ConsoleLogger) Error(msg string) {
    fmt.Printf("[ERROR] %s\n", msg)
}

// Implement for file
type FileLogger struct {
    filename string
}

func (fl FileLogger) Info(msg string) {
    // Write to file
    fmt.Printf("[FILE-INFO] %s\n", msg)
}

func (fl FileLogger) Error(msg string) {
    fmt.Printf("[FILE-ERROR] %s\n", msg)
}

// Function accepts any Logger implementation
func ProcessDeploy(logger Logger) {
    logger.Info("Starting deployment")
    // ... deployment logic ...
    logger.Error("Connection timeout")
}

func main() {
    // Use ConsoleLogger
    consoleLog := ConsoleLogger{}
    ProcessDeploy(consoleLog)

    // Use FileLogger
    fileLog := FileLogger{filename: "/var/log/deploy.log"}
    ProcessDeploy(fileLog)
}
```

### 5. Goroutines and Channels

```go
package main

import (
    "fmt"
    "time"
)

// goroutines: lightweight threads
// channels: safe communication between goroutines

func checkServer(name string, result chan string) {
    // Simulate server check
    time.Sleep(1 * time.Second)
    result <- fmt.Sprintf("%s is healthy", name)
}

func main() {
    // Create channel for results
    results := make(chan string, 3)

    // Start multiple goroutines
    servers := []string{"web01", "web02", "db01"}
    for _, server := range servers {
        go checkServer(server, results)  // Don't wait, continue
    }

    // Collect results
    for i := 0; i < len(servers); i++ {
        msg := <-results  // Receive from channel
        fmt.Println(msg)
    }
}
```

This demonstrates Go's true concurrency where multiple operations run simultaneously.

### 6. CLI with Cobra

```go
// CLI tool framework (like Click in Python)

package main

import (
    "fmt"
    "os"
    "github.com/spf13/cobra"
)

var rootCmd = &cobra.Command{
    Use:   "devops",
    Short: "DevOps automation CLI",
    Long:  "Complete DevOps automation tool",
}

var deployCmd = &cobra.Command{
    Use:   "deploy",
    Short: "Deploy application",
    Run: func(cmd *cobra.Command, args []string) {
        env, _ := cmd.Flags().GetString("env")
        fmt.Printf("Deploying to %s...\n", env)
    },
}

var statusCmd = &cobra.Command{
    Use:   "status",
    Short: "Check service status",
    Run: func(cmd *cobra.Command, args []string) {
        fmt.Println("All services running")
    },
}

func init() {
    deployCmd.Flags().StringP("env", "e", "staging", "Deployment environment")
    rootCmd.AddCommand(deployCmd)
    rootCmd.AddCommand(statusCmd)
}

func main() {
    if err := rootCmd.Execute(); err != nil {
        fmt.Println(err)
        os.Exit(1)
    }
}
```

## 🏢 In Practice

### Kubernetes Client-go Example

```go
package main

import (
    "context"
    "fmt"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/client-go/kubernetes"
    "k8s.io/client-go/tools/clientcmd"
)

func main() {
    // Load kubeconfig
    config, err := clientcmd.BuildConfigFromFlags("", clientcmd.RecommendedHomeFile)
    if err != nil {
        panic(err)
    }

    // Create clientset
    clientset, err := kubernetes.NewForConfig(config)
    if err != nil {
        panic(err)
    }

    // List pods in all namespaces
    pods, err := clientset.CoreV1().Pods("").List(context.Background(), metav1.ListOptions{})
    if err != nil {
        panic(err)
    }

    for _, pod := range pods.Items {
        fmt.Printf("Pod: %s/%s\n", pod.Namespace, pod.Name)
    }
}
```

## ⚠️ Common Mistakes

### Mistake 1: Not Checking Errors

```go
// ❌ Wrong — silent failures
data := ioutil.ReadFile("config.yaml")

// ✅ Correct — check errors
data, err := ioutil.ReadFile("config.yaml")
if err != nil {
    log.Fatalf("Failed to read config: %v", err)
}
```

### Mistake 2: Not Using Pointers for Large Structs

```go
// ❌ Inefficient — copies entire struct
func ProcessPod(p Pod) {
    // ...
}

// ✅ Efficient — passes reference
func ProcessPod(p *Pod) {
    // ...
}
```
